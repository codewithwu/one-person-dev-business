# 收钱的艺术

## 定价是最被低估的技能

大多数程序员对定价的态度是：随便定一个，看看再说。

这是一个昂贵的错误。定价直接决定了你的收入，而大多数独立开发者的定价都偏低。

一个产品月付 $9 和月付 $29 的区别，不只是三倍的收入——它还决定了你能不能持续做下去。$9 的定价可能让你入不敷出最终放弃，$29 的定价可能让你体面地活下来。

## 怎么定价

### 看竞品

先看看类似产品卖多少钱。你的价格不需要和竞品一样，但需要一个参照系。

如果你的产品更好，可以定更高。如果你是新入场的，可以先低一些获取用户再涨价。

### 订阅 vs 一次性

**订阅（推荐大多数场景）：**
- 收入可预测，现金流稳定
- 用户门槛低（月付 $10 比一次性 $120 好卖得多）
- 用户留存 = 持续收入

**一次性付费：**
- 适合模板、主题、课程等一次性内容
- 收入不稳定，需要持续获取新用户

**混合：** 免费层级 + 付费订阅（Freemium）是最常见的 SaaS 模式。关键在于免费版本要足够好用能吸引人，但也要有足够强的理由让用户升级。

### 定价心理学

- 以 9 结尾的价格在心理上感觉便宜很多。$29 比 $30 好卖。
- 提供三个价格选项，人倾向于选中间那个。把你想卖的版本放在中间。
- 年付打折是双赢。用户省钱，你提前拿到一年收入。
- 涨价比降价容易。老用户可以锁旧价，新用户按新价。

## 增长

产品做好了，定价也定了，怎么让更多人知道？

### 产品驱动增长

让产品本身成为获客工具。

- 免费层级让用户零成本开始用
- 协作功能让用户拉其他人一起用
- 用户创建的内容本身有 SEO 价值
- 「Powered by XXX」的小水印带来自然曝光

### 内容驱动增长

通过有价值的内容吸引潜在用户。

- 写解决目标用户痛点的博客文章
- 发布教程和案例研究
- 做 YouTube 视频或播客

内容的好处是长期有效。一篇好文章可以持续带来流量好几年。

### 社区驱动增长

在目标用户聚集的地方提供价值。

- Reddit、Hacker News、V2EX 相关板块
- Discord / Slack 群组
- GitHub Discussions

关键是先提供价值，再提及产品。不要一上来就卖东西。

## 规模化

一人公司的规模化不是招更多人，而是用更少的时间做更多的事。

**自动化一切。** 客服用 AI 处理常见问题。部署用 CI/CD。账单用 Stripe 自动生成。

**标准化。** 把重复工作变成流程。客户 onboarding、bug 修复、功能发布——每个流程有 checklist。

**优化而非扩张。** 提高转化率、降低流失率、提高客单价——这些比拓展新渠道更有效。

**知道什么时候该停。** 一人公司的目标不是无限增长。收入到了满意的水平，就维持和优化，而不是继续扩张。
