# 从零到一的完整流程

## 一条可复制的路径

这一章把前面的理论串起来，走一遍从想法到上线的完整流程。每一步都尽量具体，让你看完就能动手。

```
第1周：找方向 + 验证需求
第2-3周：做 MVP
第4周：上线 + 获取第一批用户
之后：持续迭代
```

总时间：一个月左右。用 AI 辅助的话可以更快。

## 第一周：找方向

### Day 1-2：列出候选方向

花两天时间，列出 5-10 个你觉得值得做的方向。来源：

- 你自己的痛点和需求
- 社区里的抱怨和 feature request
- 竞品的差评和不足
- 你观察到的行业趋势

每个方向用一句话描述：「给谁解决什么问题」。

### Day 3-4：筛选

用前面讲的标准筛一遍：

- 够痛吗？（用户愿意付费吗）
- 够大吗？（至少几千潜在用户）
- 能做吗？（技术可行，一个人能搞定）
- 有优势吗？（你的经验、技术栈匹配吗）

筛到 1-2 个最靠谱的方向。

### Day 5-7：验证

用最低成本验证需求。

**做落地页。** 用 v0 十分钟生成一个产品介绍页。写清楚：产品是什么、解决什么问题、定价多少。加上一个「加入等待列表」或「立即购买」的按钮。

**投放。** 把落地页发到目标社区（Reddit 相关 subreddit、V2EX、即刻、Twitter）。看有多少人点击、留邮箱、或者直接付费。

**直接聊。** 找 5-10 个目标用户聊。不需要正式的访谈，就是聊天：你现在怎么解决这个问题？花多少钱？对现有方案哪里不满意？

**判断标准：** 如果落地页的转化率超过 2%，或者有 3 个以上的人说「我现在就需要」，方向基本可以确定。

## 第二到三周：做 MVP

### 范围控制

MVP 的关键词是「最小」。只做核心功能，其他全部砍掉。

问自己：如果用户只能用一个功能，是哪个？只做那个。

**界面：** 用 v0 生成 React 组件，能用就行。不要纠结配色和间距。

**后端：** 用 Supabase 做数据库和认证。用 Vercel 或 Railway 部署。能跑就行。

**支付：** 接 Stripe 或 LemonSqueezy。能收款就行。

**AI 辅助开发：** 用 Cursor 或 Claude Code 写代码。它写初稿，你审查修改。遇到不熟悉的技术栈，直接问 AI。

### 开发节奏

- 第 1-3 天：搭建项目骨架、数据库设计、认证系统
- 第 4-7 天：核心功能开发
- 第 8-10 天：接入支付、写产品文案、做落地页
- 第 11-14 天：测试、修 bug、部署上线

**一个提醒：不要在 MVP 阶段追求完美。** 代码不够优雅？没关系。UI 不够精致？没关系。先上线，看用户反应，再迭代。

## 第四周：上线

### 部署

- 前端：Vercel（git push 自动部署）
- 后端：Railway 或 Fly.io
- 数据库：Supabase 或 Neon
- 域名：Namecheap 买域名，Cloudflare 做 DNS

### Product Hunt 发布

Product Hunt 是全球最大的新产品发布平台。一个好的 launch 可以带来几千甚至几万的初始用户。

**准备清单：**
- 产品名称和 tagline（简短、有记忆点）
- 产品截图和演示视频
- 第一批评论（找朋友帮你在发布日留言）
- 发布时间：太平洋时间凌晨 12:01（这是 Product Hunt 的每日重置时间）

### 社区分享

在目标社区分享你的产品。关键是提供价值，不是发广告。

**正确的做法：** 「我做了这个工具来解决 X 问题，分享给大家。欢迎试用和反馈。」

**错误的做法：** 「我的产品超棒，快去买。」

### Building in public

在 Twitter/X 上分享你的开发过程、产品数据、用户反馈。独立开发者社区对这种透明的分享方式非常友好。它本身就是一种营销。

## 上线之后

**第一周：** 密切关注用户反馈。回复每一封邮件。快速修 bug。不要急着加新功能。

**第一个月：** 收集用户行为数据。看用户怎么用你的产品、哪些功能用得多、哪些没人用。根据数据调整优先级。

**持续：** 小步快跑。每周一个小更新，每月一个大更新。让用户看到产品在进步。
