# AI 时代的武器库

## 你不是一个人在战斗

2026 年做一人公司，你的团队配置是：你 + 若干 AI 工具 + 云服务。

这个「团队」能做到的事比五年前一个三四人的小团队还多。前提是你要知道有哪些武器，以及怎么组合使用。

## 写代码

**Cursor** —— 当前最流行的 AI 编程 IDE。基于 VSCode，深度集成了大模型能力。你可以用自然语言描述需求，它帮你生成代码、重构、调试。特别适合快速原型和全栈开发。

**Claude Code** —— Anthropic 出品。擅长理解复杂代码库、跨文件重构、架构级修改。适合有一定规模的项目，或者需要做深度代码变更的场景。

**GitHub Copilot** —— 最老牌的 AI 编程助手，嵌入在 VSCode 和 JetBrains 中。行内补全能力强，适合日常编码。

**怎么选？** 不用纠结，都试试。Cursor 适合快速开发，Claude Code 适合复杂项目，Copilot 适合日常补全。很多人三个都用，不同场景切换。

**一个提醒：AI 写的代码一定要 review。** 它能跑，但不一定最优。可能会有安全漏洞、性能问题、或者不符合你的架构规范。把 AI 当成一个干活很快但需要你把关的初级工程师。

## 做设计

**v0 (Vercel)** —— 用自然语言描述 UI，直接生成 React 组件。做出一个能看的产品原型，十分钟的事。

**Figma AI** —— 设计稿自动生成、智能布局、组件建议。不会专业设计也能做出 70 分的界面。

**Midjourney / DALL-E** —— 图片生成。做 logo、营销素材、社交媒体配图。

**现实一点说：** 大部分程序员的审美是够用但不出彩。AI 生成的设计也是「够用但不出彩」。两者结合，做出来的东西至少不丢人。如果你的产品需要极致的设计感，还是得找设计师。但 MVP 阶段，AI 足够了。

## 写内容

**Claude / GPT-4** —— 写产品文档、营销文案、技术博客的初稿。AI 初稿需要人工润色，否则读起来有明显的「AI 味」。

**DeepL / Claude** —— 翻译。做海外市场必备。

**HeyGen / Synthesia** —— AI 视频生成。不用真人出镜也能做产品演示视频。适合不想露脸但需要视频内容的开发者。

## 运营和客服

**Crisp / Intercom** —— AI 客服机器人。自动回复用户常见问题，你只需要处理复杂case。

**Resend** —— 开发者友好的邮件发送服务。发通知邮件、营销邮件、产品更新。

**PostHog** —— 产品分析。追踪用户行为、转化漏斗、功能使用率。自托管免费。

## 基础设施

**Vercel** —— 前端部署。Git push 自动部署，自带 CDN 和 SSL。

**Railway / Fly.io** —— 后端部署。适合需要运行服务端代码的场景。

**Supabase** —— 开源的 Firebase 替代。数据库 + 认证 + 存储，免费层级够用。

**Cloudflare** —— DNS、CDN、安全。Workers 可以跑 serverless 函数，免费额度很大。

**Stripe / LemonSqueezy** —— 支付。Stripe 功能最全，LemonSqueezy 对独立开发者更友好（帮你处理税务和发票）。

## 选择原则

**够用就好。** 你是一人公司，不是大厂架构组。能用免费层级的就不要付费。能用一个工具解决的就不要用三个。

**优先选有 API 的。** 有 API、有 CLI、有 SDK 的工具才能被你自动化。没有 API 的工具是在给你增加手动工作量。

**数据要能导出。** 不要锁死在某个平台上。选工具前确认数据能不能迁移。

**别过度工程化。** MVP 阶段用最简单的方案。等产品验证了、用户增长了，再考虑换更专业的工具。
